# コーディング規約

SIVIRA 共通規約（`~/.claude/specs/coding-standards.md`）に加え、本プロジェクト固有のルールを定める。

---

## 言語・ツールチェイン

- **Rust** stable（最新の安定版）
- フォーマット: `cargo fmt`（`rustfmt.toml` はデフォルト設定）
- Lint: `cargo clippy -- -D warnings`（警告をエラーとして扱う）
- テスト: `cargo test`

---

## デバッグコードの残留禁止

`dbg!`、`println!`（デバッグ目的）、`todo!`、`unimplemented!` を本番コードに残さない。

**正しい方法:** `tracing` クレートを使い、`info!`、`warn!`、`error!` 等のマクロでログレベルを制御する。

---

## エラーハンドリング

- `unwrap()` / `expect()` は、値が存在しないことが論理的にありえない場合のみ使用する。その場合はコメントで理由を明記する。
- 外部 I/O（Slack API、Vegapunk gRPC、ファイルシステム）は `Result` で伝播し、呼び出し元で適切にハンドリングする。
- `anyhow` または `thiserror` でエラー型を整理する。

---

## 型安全・バリデーション

- Slack API のレスポンス、環境変数など外部入力は使用前にバリデーションする。
- `unsafe` ブロックは原則禁止。やむを得ない場合は理由をコメントで明記する。

---

## マジックナンバー・マジックストリングの排除

- 定数は `const` または設定ファイルで管理する。
- Slack API の URL、gRPC メソッド名などのリテラルは一箇所にまとめる。

---

## 依存パッケージのバージョン固定

- `Cargo.lock` をコミットする（バイナリプロジェクトのため）。
- `Dockerfile` のベースイメージはタグを固定する（`rust:1.XX-slim` 等）。

---

## Docker

- ホストに直接依存する操作は行わない。ビルド・テスト・実行はすべてコンテナ内で完結させる。
- マルチステージビルドでイメージサイズを最小化する。

---

## `.gitignore` の徹底

SIVIRA 共通規約に準拠する。Rust 固有の除外対象:

- `target/`
- `**/*.rs.bk`
